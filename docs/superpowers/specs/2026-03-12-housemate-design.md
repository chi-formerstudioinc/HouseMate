# HouseMate — Design Spec
**Date:** 2026-03-12
**Status:** Approved

---

## Overview

HouseMate is an iOS app for couples and roommates to manage their shared household. It covers three core areas: task and chore management, waste and recycling reminders, and home maintenance tracking. The app is built with SwiftUI and CloudKit, requiring no custom backend.

---

## Target Audience

Couples or roommates sharing a home who want a simple, shared system to coordinate household responsibilities.

---

## Platform

- iOS only (SwiftUI)
- Testable on physical device via Xcode without App Store submission
- CloudKit development environment available during testing at no cost
- App Store publishing requires Apple Developer account ($99/year) — deferred to a later phase

---

## Architecture

### Technology Stack
- **Frontend:** SwiftUI (iOS)
- **Data & Sync:** CloudKit (shared zone for household data, private zone for user preferences)
- **Push Notifications:** CloudKit subscriptions (member activity) + UNUserNotificationCenter (scheduled reminders, per-device)
- **Auth:** iCloud identity via CloudKit (no separate auth system needed)

### CloudKit Conventions
All cross-record references are `CKRecord.Reference` types using `.deleteSelf` action unless noted. All models have an implicit CloudKit record ID (`CKRecord.ID`) as their primary identifier. `templateID` fields on `Task` and `MaintenanceItem` store the `CKRecord.ID.recordName` string of their source template.

### CloudKit Zone Structure
- **Private zone** — user profile, display name, notification preferences
- **Shared zone** — household record, tasks, bin schedule, maintenance items, maintenance logs, task completion logs (owned by household creator, shared with invited members via CloudKit share)

### Household Invite Flow
1. User creates a household → CloudKit generates a `CKShare` URL
2. That URL is surfaced in the app as a shareable link/code
3. Invited member taps the link on their device → accepts the CloudKit share → gains read/write access to the shared zone
4. Both devices sync automatically via CloudKit
5. The invite link can be regenerated by the household creator at any time (invalidating the previous link)
6. A household has a maximum of 6 members
7. The same link can be used by multiple people up to the member limit

### Local Notification Distribution
Bin day and maintenance reminders use `UNUserNotificationCenter` (local, per-device). Because `BinSchedule` and `MaintenanceItem` records live in the shared CloudKit zone, every member's device has access to the same schedule data. Each device independently schedules its own local notifications on first launch and whenever the relevant records change (observed via CloudKit subscriptions). This means all members receive these reminders without requiring server-side push.

**Recurring task lifecycle:**
- A recurring task is stored as a single `Task` record. There are no pre-generated future instances.
- When a member completes a recurring task, the app: (1) creates a `TaskCompletionLog` entry, (2) resets `isCompleted` to false, (3) advances `dueDate` by the `recurringInterval`, and (4) clears `completedBy` / `completedAt`.
- If `dueDate` is nil when a recurring task is completed, the next due date is set to `today + recurringInterval`.
- **Concurrent completion (two members complete simultaneously):** Both `TaskCompletionLog` entries are created. The first write to CloudKit wins for date advancement. The second member's device, upon syncing, detects the task was already completed (via server timestamp check) and shows an alert: "This task was already completed." No duplicate date advancement occurs.
- To prevent duplicate generation when two members are online simultaneously, completion writes use CloudKit atomic saves with a server-side timestamp check. The last write wins; the UI reconciles on the next sync.
- When `isRecurring` is toggled off on an existing recurring task, `recurringInterval` is set to nil and `dueDate` is left as-is (the current due date).

### TaskCompletionLog
| Field | Type | Notes |
|---|---|---|
| taskID | CKRecord.Reference | Parent task |
| completedBy | CKRecord.Reference | Member who completed it |
| completedAt | Date | |

Task detail view shows the last 5 `TaskCompletionLog` entries for that task, ordered by `completedAt` descending.

**Task deletion:** Deleting a task hard-deletes the `Task` record and all associated `TaskCompletionLog` records (via `.deleteSelf` reference action). This is intentional — completion history is not retained after task deletion. The confirmation alert reads: "Delete this task? Its completion history will also be deleted." with Cancel / Delete actions.

**Template storage:** Built-in templates are bundled with the app and stored locally (not in CloudKit). User-created templates are stored in the shared CloudKit zone so all household members can see and use them.

### BinSchedule
**Cardinality:** Exactly one `BinSchedule` record per household. If none exists, the Bins tab shows an empty state with a "Set Up Bin Schedule" button that opens the Configure Schedule sheet directly.

| Field | Type | Notes |
|---|---|---|
| pickupDayOfWeek | Int | 1 = Sunday … 7 = Saturday, matching Swift `Calendar.weekday`. Default: 5 (Thursday) |
| rotationA | String | User-facing label for even-week pickup, e.g. "Compost + Recycling". Max 50 chars. Required. |
| rotationB | String | User-facing label for odd-week pickup, e.g. "Compost + Garbage". Max 50 chars. Required. |
| startingRotation | Enum | A or B — the rotation that applies to the `startingDate` week |
| startingDate | Date | A known past pickup date used as the reference anchor for rotation calculation |
| notifyDayBefore | Bool | |
| notifyMorningOf | Bool | |

**Rotation calculation algorithm:**
Given a target pickup date `D`, rotation is determined as:
1. Compute the number of weeks between `startingDate` and `D`: `weeksDiff = floor((D - startingDate) / 7)`
2. If `weeksDiff` is even → use `startingRotation`; if odd → use the other rotation
3. `startingDate` must be a past or present date; the UI validates this on save
4. If the user changes `pickupDayOfWeek`, `startingDate` resets to the most recent past occurrence of the new pickup day, and `startingRotation` must be re-confirmed by the user

### MaintenanceItem
| Field | Type | Notes |
|---|---|---|
| name | String | |
| category | Enum | Spring, Summer, Fall, Winter, Year-Round |
| intervalDays | Int | |
| lastCompletedDate | Date? | Optional — nil for items never completed |
| notes | String? | Optional — permanent item-level notes |
| templateID | String? | Optional — stores source MaintenanceTemplate record name |

**Next due date calculation:** `lastCompletedDate + intervalDays`. If `lastCompletedDate` is nil, the item is shown as "Not yet completed" and treated as overdue for dashboard and color-coding purposes.

**Color-coding thresholds (unified across Maintenance tab and dashboard):**
- Green: next due date is more than 14 days away
- Yellow: next due date is within 14 days (including today)
- Red: next due date is in the past (overdue), or `lastCompletedDate` is nil

**Dashboard "Maintenance Due Soon" card:** shows all items with yellow or red status (due within 14 days or overdue).

## Screens & Navigation

### Tab Bar (4 tabs)
1. Home
2. Tasks
3. Bins
4. Maintenance

Plus a Settings screen accessible via a profile/gear icon in the Home tab navigation bar.

---

### Onboarding (first launch)

**Happy path:**
- Check for iCloud sign-in; if not signed in, show an informational screen explaining why iCloud is required with a "Go to Settings" button — app cannot proceed without it
- Choice: **Create a Household** or **Join a Household**
  - Create → enter household name → CloudKit share created → invite link shown with copy/share options
  - Join → paste or tap invite link → accept CloudKit share → synced to shared zone → proceed to Home

**Error states:**
- iCloud unavailable (offline, account issue): show persistent banner "HouseMate requires iCloud. Check Settings > [Your Name] > iCloud."
- Invite link invalid or already at member limit: show alert "This invite link is no longer valid. Ask your household admin for a new one."
- CloudKit temporarily unavailable (network error): show alert with retry option; do not block app if user is already a member with cached data

---

### Home (Dashboard)

- Greeting: "Good morning, [displayName]"
- **Today's Tasks card** — tasks assigned to the current user with `dueDate` = today (compared using the device's local timezone); tap to complete inline. Empty state: "No tasks for today — enjoy your day!"
- **Next Bin Day card** — days until next pickup + rotation label (e.g. "Recycling & Compost in 3 days")
- **Maintenance Due Soon card** — items with yellow or red status (due within 14 days or overdue)
- **Quick-add FAB** — floating action button to add a task (opens a compact sheet: title, category, optional due date, optional assignee)

---

### Tasks Tab

**List view:**
- All household tasks, default sort: overdue first, then by due date ascending, then undated
- Filter bar: All / Mine / Unassigned / Completed
- Overdue tasks shown with a red indicator
- Swipe right to complete; swipe left reveals a Delete button (requires confirmation alert: "Delete this task?" with Cancel / Delete actions — hard delete, removes for all members)
- Tap task to view detail

**Task detail:**
- Title, category, priority, assignee, due date, recurring settings
- Last 5 completion log entries (date + who completed it), sourced from `TaskCompletionLog`
- Edit button

**Add/Edit task form:**
- Title (required), category, priority, assign to member (picker), due date (date picker), recurring toggle → interval picker (Daily / Weekly / Monthly)

**Templates:**
- "Browse Templates" button opens a sheet grouped by category
- Built-in templates listed; user-created templates listed separately
- Tap to instantiate as a new task (pre-fills the Add form)
- Long-press template → option to delete (user-created only)

**Notifications:**
- Task completed by a member → other members receive: "[Name] completed '[Task title]'"
- Task assigned to a member → that member receives: "[Name] assigned you '[Task title]'" — fires on both initial assignment and reassignment; the previously assigned member is not notified of the reassignment in v1

---

### Bins Tab

- **Current rotation card** — today's date, which rotation is next, and how many days away
- **Upcoming pickups list** — next 8 pickup dates with rotation label for each
- **Configure Schedule button** → settings sheet:
  - Pickup day of week (day picker)
  - Rotation A label (text field, default "Compost + Recycling")
  - Rotation B label (text field, default "Compost + Garbage")
  - Starting rotation selector (A or B) with starting date picker
  - Notify: day before toggle, morning-of toggle
  - Save → reschedules all local notifications for the coming 12 months

> Municipality calendar import is explicitly deferred to a future version. No UI placeholder is included in v1.

---

### Maintenance Tab

**List view:**
- Items grouped by seasonal category (Spring, Summer, Fall, Winter, Year-Round)
- Each row: name, last done date (or "Never"), next due date, color-coded status dot
- Tap item → item detail

**Item detail:**
- Name, category, interval, next due date, color status
- Permanent notes field (editable inline)
- **Log Completion button** → sheet: date (default today), notes (optional), cost (optional decimal field)
- Full history log: all `MaintenanceLog` entries ordered by `completedDate` descending
- Edit item button (name, interval, category)

**Add item:**
- Manual entry: name, interval (days), seasonal category, last completed date (optional)
- Or tap "From Templates" → browse built-in and custom templates → tap to pre-fill the form

**Built-in maintenance templates:**
- Change furnace filter — 90 days — Year-Round
- Replace HVAC filter — 90 days — Year-Round
- Clean dryer vent — 365 days — Year-Round
- Sweep/blow out garage — 30 days — Year-Round
- Test smoke detectors — 180 days — Year-Round
- Clean range hood filter — 90 days — Year-Round
- Flush water heater — 365 days — Year-Round
- Check window/door seals — 365 days — Fall
- Clean gutters — 180 days — Spring
- Winterize outdoor faucets — 365 days — Fall

**Maintenance notifications:**
- Scheduled local notification fires on the item's next due date at 9 AM (device local time): "Time to: [item name]"
- After each log entry is saved, the local notification for that item is cancelled and rescheduled based on the new `lastCompletedDate + intervalDays`
- **Notification budget:** iOS caps local notifications at 64 per app. To stay within this limit, each maintenance item schedules only its single next notification (not recurring future ones). Bin day notifications consume at most 2 slots (day-before and morning-of for the next pickup). On app launch and after any relevant CloudKit change, all maintenance and bin notifications are cancelled and rescheduled from scratch.

### Settings

- Household name (editable, saves to CloudKit)
- Members list with display names
- **Invite Member** — shows current share link with copy/share options; **Regenerate Link** button deletes the existing `CKShare` and creates a fresh one, invalidating the previous link
- **Member removal** — not supported in v1. Members can leave a household via iCloud Settings > [Your Name] > iCloud > Shared with You. This is noted in the Settings screen with a help text: "To remove a member, ask them to leave via iCloud Settings."
- Notification preferences: per-type toggles (task completion, task assignment, bin day before, bin day morning, maintenance due)
- Your display name (editable)

## Notification Summary

| Trigger | Delivery | Recipient |
|---|---|---|
| Task completed by member | CloudKit subscription → remote push | Other household members |
| Task assigned to you | CloudKit subscription → remote push | Assigned member |
| Bin day (day before, 6 PM) | Local scheduled (per device) | All members independently |
| Bin day (morning of, 7 AM) | Local scheduled (per device) | All members independently |
| Maintenance item due | Local scheduled (per device, fires at 9 AM on due date) | All members independently |

Local notifications are rescheduled on app launch and whenever a CloudKit subscription fires indicating a change to `BinSchedule` or `MaintenanceItem` records.
